## Learning Objectives

By the end of this seminar, participants will be able to:

1. **Install and configure** Git on their local machine
2. **Create repositories** and track changes with commits
3. **Push and pull** code to/from GitHub
4. **Create branches** and merge changes
5. **Collaborate** using pull requests and code reviews
6. **Apply best practices** for version control in research and development

## How to Use This Tutorial

- **As a speaker:** Follow the [Speaker Slide Outline](speaker_slides_outline.md) for presentation flow and talking points. Use the [demo/](demo/) folder for live coding demonstrations.
- **As a participant:** Work through each part sequentially. Each file contains explanations, commands to run, and exercises.
- **As a self-paced learner:** Follow each part at your own speed. All commands are copy-paste ready.

## Audience

This seminar is designed for:
- Graduate students and researchers
- Software developers new to version control
- Anyone who wants to collaborate more effectively on code or documents
